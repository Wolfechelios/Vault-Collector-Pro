# WolfeVault 0.7 — Integrated Inventory Intelligence

## Included systems

- Evidence-backed OCR, barcode, logo, object, identifier, attribute, condition, and category extraction.
- Deterministic confidence routing: high auto-applies, medium auto-applies with a visible flag, low/conflicting/protected replacements require review.
- Protected manual fields and full source evidence provenance.
- Data-driven category-specific fields.
- Local correction learning with inspectable alias, category, storage, provider-routing, and title-format rules.
- Rule influence identifiers on every affected suggestion.
- Offline FTS5 search with deterministic natural-language parsing and structured filters.
- Saved searches, search history, smart collections, and cards/table/messy-closet result views.

## Persistence

Migration `0005_inventory_intelligence.sql` creates evidence, suggestions, field state, category definitions, correction rules, learning events, search documents, FTS5, background reindex queue, saved searches, and search history. The migration uses a transaction, is safe to re-run, backfills existing items into the reindex queue, and never replaces current item fields during initialization.

## Protection and privacy

- User-entered and verified values are never replaced automatically.
- All rules remain local, readable, editable, disableable, and removable.
- No private inventory is uploaded for model training.
- Search and all core intelligence remain functional offline.

## Release gate

The pull request must pass all TypeScript unit/UI tests, desktop and mobile production builds, SQLite migration/FTS tests, Rust repository tests, Rust formatting, Clippy, and GitHub Actions. The release must not merge with a failing check.
