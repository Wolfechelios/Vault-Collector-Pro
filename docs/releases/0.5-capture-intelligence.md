# Vault 0.5 Capture Intelligence

## Delivered as one integrated release

- Shared `@vault/capture` engine used by desktop and mobile.
- Persistent resumable background-job model with retries, cancellation, progress and queue summaries.
- Six-digit expiring one-time phone pairing sessions and device identity.
- Offline phone capture queue supporting up to 24 photos per item.
- Local capture-bundle export/import with item and photo metadata.
- Photo quality scoring for resolution, sharpness and exposure.
- Perceptual hashing, Hamming distance and duplicate-confidence matching.
- Category, brand and model inference from labels, filenames and notes.
- Desktop Capture Center with queue controls, pairing panel, photo workspace and duplicate review.
- SQLite tables for capture jobs, item photos, paired devices, capture sessions and duplicate matches.
- Catalog integration that converts phone captures into real private inventory records.
- CI coverage for capture tests, desktop/mobile production builds and Rust migrations.

## Data and privacy

Core capture remains local-first. Phone bundles are local JSON transfers containing the user's selected photos. No cloud service or external account is required. Pairing tokens, catalog databases, personal media and API credentials must never be committed.

## Verification gate

This release is mergeable only when all workspace tests pass, desktop and mobile production builds pass, and Rust/SQLite tests pass on macOS.
